# 要求内容

## 概要

6時間を超えるイベントの登録をスキップするバリデーション処理を本番コードに追加する。

## 背景

- 過去に `feature/reject-long-events` ブランチで旧 dev 環境（`コード.js`）に6時間超過チェックを実装していた
- その後、本番コードが大幅にリアーキテクト（メール集約送信化、ブラックリスト機能追加等）された際に、この処理が取り込まれなかった
- 結果として、現在の本番コード（`createOrUpdateCalendarEventC.js`）には6時間チェックが欠如しており、6時間超のイベントがカレンダーに登録されてしまう

## 実装対象の機能

### 1. 6時間超過イベントの登録ブロック

- `createOrUpdateCalendarEvent()` 内で、開始日時と終了日時の差が6時間を超える場合にカレンダー登録を行わない
- 既存のイベントIDがある場合（更新時）はカレンダーからイベントを削除し、IDをクリアする
- 変更フラグを立て、送信済みフラグをクリアして `sendAggregatedEmails()` での通知対象にする

## 受け入れ条件

- [ ] 終了日時 - 開始日時 > 6時間の場合、カレンダーにイベントが登録されない
- [ ] 既存イベントが存在する場合（更新時）、カレンダーから削除される
- [ ] 変更フラグが立ち、送信済みフラグがクリアされる（メール通知の対象になる）
- [ ] `sendAggregatedEmails()` で6時間超過のケースが正しく処理される（通知メール文面）
- [ ] 他のバリデーション（過去日時チェック、逆転チェック）に影響しない

## 成功指標

- 6時間超のイベントがカレンダーに登録されないこと

## スコープ外

- dev 環境の再構築
- メール文面のデザイン変更
- 6時間の閾値の変更
